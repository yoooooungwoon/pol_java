# 2026-06-09 클래스

## 오늘 배운 내용

- 함수는 여러 줄의 코드를 하나의 이름으로 묶어 재사용할 수 있게 해준다.
- Java에서는 클래스 안에 함수를 작성하며, 보통 메서드라고 부른다.
- 함수는 입력값을 받을 수 있고, 실행 결과를 반환할 수 있다.
- 같은 동작을 여러 번 작성하지 않고 함수로 분리하면 코드가 더 읽기 쉬워진다.

## 기억할 것

- 생성자는 한개가 아니라 여러개 존재할 수 있다.
- 생성자에서 다른 생성자를 호출해서 사용할 수 있다.
- 오버로딩은 변수명이 아니라 매개변수의 타입, 개수, 순서만 본다
-> 추가적인 설명으로 
  public Cleric(String name, int hp){
  this(name, hp, MX_MP);
  }
  public Cleric(String name, int mp){
  this(name, hp, MX_MP);
  } 이런경우에는 뭔지 판단을 할수 없다. 매개변수의 타입, 개수, 순서로만 판단하기 때문이다.
## 실습 코드

```java
    public Cleric(String name, int hp){
        this(name, hp, MX_MP);
}
    public Cleric(String name, int mp) { // 컴파일 오류(동일 시그니처) 예시
        this(name, MX_HP, mp);
}
```

## 어려웠던 점

- 이 클래스는 new Cleric() 과 같이 이름을 지정하지 않는 경우에는 인스턴스화 할 수 없다고 한다. (이름이 없는 성직자는 존재 할 수 없음) 
- 위의 문제를 생각을 잘 못했는데 실제로 생성자를 만들면 자동으로 생성되지 않기때문에 이미 적용이 된 것이다.
